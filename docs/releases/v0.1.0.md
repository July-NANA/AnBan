# Anban v0.1.0 Release Notes

Status: released on 2026-07-17 from exact SHA
`e1fc0845581460303d2b1971f21bdece9b34de93`; Gate #28, tag `v0.1.0`, GitHub Release,
`main` exact-SHA CI, and Milestone closure completed.

v0.1 delivers one real fixed Agent Runtime with two production Capabilities: `skill.activate` and
`process.execute`. All package and Workspace Skills use the same discovery, parsing, activation,
execution, and persistence architecture. The packaged `@anban/clawhub` contains CLI knowledge but
has no Runtime special case. A newly installed Skill becomes available after a new Application
discovers its `SKILL.md`.

Process can execute PATH or absolute programs with inherited environment, optional env/cwd/stdin,
bounded output and timeout/cancellation, process-group cleanup, stable failure semantics, safe
summary Metadata, and multiple explicitly declared single-file Artifacts per invocation. Native
Tool Calls remain authoritative when a Provider includes non-authoritative companion text.
Workspace `@anban/*` names are reserved, duplicate ordinary slugs are all excluded, and Skill
version/source records do not enter production identity. PostgreSQL preserves complete
Invocation/Event correlations and restart-safe Audit/Trace projections.

Scoped plain-Markdown Skills derive identity from their path, while unscoped frontmatter display
names are normalized deterministically. A bounded ordinary Tool failure observation can be returned
to the model for recovery without changing the failed Invocation fact. Stable pre-execution
argument and availability failures follow the same path after the failed Invocation is durable.
Terminal persistence uses commit confirmation and one compensation transaction without replay;
confirmed uncommitted managed snapshots are cleaned while sources remain untouched. Final
user-visible results may report legitimate paths; Error and Event Metadata retain the stricter
physical-path boundary.

Known limitation: v0.1 has no program allowlist, sandbox, approval workflow, network isolation, or
fine-grained file permissions. It uses the launching OS user's authority. Browser, MCP, additional
Model Ports, persistence backends, dynamic graphs, multi-Agent orchestration, replay, and routing
remain outside v0.1.
