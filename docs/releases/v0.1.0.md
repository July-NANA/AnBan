# Anban v0.1.0 Release Notes (candidate)

Status: release candidate pending #28 automated Gate and explicit human acceptance. No tag,
GitHub Release, or `main` integration is implied by this document.

## What v0.1 delivers

v0.1 is the first real Anban execution closure. The installed CLI creates durable Task/Run identity,
executes one fixed LangGraph General Agent through a real OpenAI-compatible ModelPort, activates the
pinned Workspace Weather Skill through CapabilityPort, performs a governed file Capability, and
returns a final result. PostgreSQL retains lifecycle and ordered Event facts; Artifact bytes remain
in the external Workspace; Audit and Trace are rebuilt from the Event stream after restart.

The public commands are `workspace init`, `run`, `chat`, `runs`, `run show`, `trace`, and
`artifacts`, with stable text and JSON forms.

## Security behavior

Configuration, model, Capability, timeout, interruption, persistence, and Event/Audit/Trace
failures remain non-success. Paths cannot escape the Run Workspace. Processes use no shell, a
caller-provided executable allowlist, bounded environment, timeout, process-group termination, and
bounded stdout/stderr. Known API key values in Provider output fail before persistence or
Capability invocation. Ambiguous side effects are not replayed.

## Acceptance scope

The release Gate runs deterministic checks and explicit real Model, pinned Skill, Capability, CLI
E2E, PostgreSQL restart, and Canary security acceptance. Ordinary CI remains deterministic and
does not receive real service credentials.

## Known limitations

v0.1 has no product web UI, dynamic graph construction, multiple Agents, branch/parallel/subgraph
execution, browser, MCP, long-term memory, Cron, Webhook, external asynchronous feedback,
checkpoint resume, complete Policy Engine, approvals, strong container sandbox, Replay, Risk or
Provenance Graph, model routing/fallback, or Skill search/install/update. Chat is bounded and not
restart-resumable. Only one approved Skill can be active. Default CLI process execution has no
enabled executable.
